# Objetivos

## Objetivo general:

- Desarrollar e implementar una plataforma web de control de asistencia para la Unidad de Soporte Técnico de la ULA-NURR, que digitalice el registro diario, la gestión de horarios y genere reportes de auditoría.

## Objetivos específicos:

**Fase de Ingeniería de Requisitos (Análisis)**

- Diagnosticar e identificar las necesidades y restricciones del proceso de control de asistencia de la Unidad de Soporte Técnico.

- Documentar los flujos de trabajo de técnicos, pasantes y la coordinación junto con sus respectivas reglas de negocio para establecer la línea base del sistema.

- Analizar y especificar los requisitos funcionales y no funcionales del proceso de control de asistencia.

**Fase de Arquitectura y Modelado (Diseño)**

- Diseñar la arquitectura del software y el modelado de estructura de datos para el almacenamiento histórico, garantizando que el diseño soporte la lógica de horarios, mecanismos de autenticación y el acceso multidispositivo.

- Diseñar prototipos de interfaz de usuario (UI/UX) adaptables a entornos web y dispositivos móviles.

**Fase de Desarrollo / Codificación**

- Desarrollar los módulos de software de la plataforma web, integrando de manera prioritaria los sistemas de autenticación digital, registro de asistencia, y generación de reportes analíticos de auditoría.

**Fase de Despliegue e Infraestructura**

- Desplegar e integrar la plataforma en el entorno de red local de la unidad, estableciendo un nodo de servidor centralizado para la base de datos y habilitando puntos de acceso e interconexión inalámbrica para los usuarios autorizados.

**Fase de Pruebas y Validación**

- Evaluar el rendimiento, la fiabilidad, la seguridad en la persistencia de datos y la usabilidad de la plataforma mediante la ejecución de un plan de pruebas funcionales y de aceptación con los usuarios finales de la unidad antes del cierre del periodo académico
