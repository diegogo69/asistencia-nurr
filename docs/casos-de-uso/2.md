### Cuadro de Escenario

|  |  |
| --- | --- |
| **Nombre:** | Gestionar Horarios |
| **Actores:** | Coordinador / Administrador |
| **Actividades:** | Configuración y definición de plantillas y bloques de horarios |
| **Detalles:** | El Coordinador o Administrador se encargará de definir, actualizar, consultar y eliminar las plantillas de horarios dentro del sistema, soportando tanto bloques fijos obligatorios (para personal contratado) como esquemas flexibles por horas requeridas (para pasantes). |
| **Eventos:** | **Agregar Plantilla:** Despliega el formulario para crear un nuevo esquema de horario. |
| | **Guardar:** Almacena la plantilla (`horarios_plantillas`) y sus bloques detallados (`horarios_detalles`). |
| | **Cerrar:** Oculta el formulario emergente sin guardar cambios. |
| | **Filtrar:** Permite buscar plantillas por nombre o tipo de horario (Fijo/Flexible). |
| | **Acciones:** Despliega las opciones de Editar y Eliminar para la plantilla seleccionada. |
| | **Editar:** Permite modificar el nombre, el tipo de horario, los bloques o la cuota de horas semanales. |
| | **Eliminar:** Remueve una plantilla de la base de datos. |
| | **Primero, Anterior, Siguiente y Último:** Opciones para la navegación en la tabla de registros. |
| **Extensiones:** | **1. Agregar Plantilla** 1.1 Muestra la interfaz emergente para ingresar el nombre de la plantilla, seleccionar el tipo (*Fijo* o *Flexible*) y especificar la cantidad de horas semanales requeridas. 1.2 Si se selecciona el tipo *Fijo*, habilita la asignación de bloques por día de la semana (Lunes a Domingo) con sus horas de entrada y salida obligatorias. |
| | **2. Guardar** 2.1 Verifica que el nombre de la plantilla no se encuentre duplicado en la base de datos. 2.2 Valida que en las plantillas de tipo *Fijo*, la hora de entrada sea menor a la hora de salida en cada bloque definido. 2.3 Si existen errores de validación, resalta los campos correspondientes en rojo y muestra una alerta. 2.4 Si los datos son correctos, guarda la plantilla y sus bloques asociados en la base de datos (`horarios_plantillas` y `horarios_detalles`). |
| | **3. Cerrar** 3.1 Cancela la operación y oculta el formulario sin conservar las modificaciones. |
| | **4. Filtrar** 4.1 Consulta la base de datos filtrando por nombre o tipo de plantilla. 4.2 Si encuentra coincidencias, muestra los registros en pantalla. 4.3 Si no existen coincidencias, muestra la tabla vacía. |
| | **5. Acciones** 5.1 Selecciona una plantilla de la tabla y habilita los botones Editar y Eliminar. |
| | **6. Editar** 6.1 Recupera y carga la información de la plantilla y sus bloques correspondientes. 6.2 Ejecuta las validaciones de coherencia de horario y no duplicidad de nombre. 6.3 Actualiza los registros en la base de datos y refresca la lista. |
| | **7. Eliminar** 7.1 Muestra un mensaje emergente para solicitar confirmación de la eliminación. 7.2 Consulta la tabla `usuarios` para verificar si existen usuarios asociados a dicha plantilla (`horario_id`). 7.3 Si hay usuarios asignados, detiene el proceso y muestra una alerta indicando que debe reasignarse el horario de esos usuarios previamente. 7.4 Si no existen usuarios vinculados, elimina la plantilla y sus detalles de la base de datos. |
| | **8. Primero, Anterior, Siguiente y Último** 8.1 Si no existen registros en la base de datos, estos botones permanecen deshabilitados. 8.2 Si existen más de 10 registros, los botones Siguiente y Último se habilitan. 8.3 Si la vista se encuentra en una página posterior a la primera, los botones Primero y Anterior se habilitan. |
