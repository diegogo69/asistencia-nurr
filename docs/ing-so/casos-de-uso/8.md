### Cuadro de Escenario

| Componente | Especificación |
| --- | --- |
| **Nombre:** | Autenticar Usuarios Administrativos |
| **Actores:** | Coordinador / Administrador, Sistema (Autenticador de Gestión)

 |
| **Actividades:** | Verificación de credenciales de seguridad (documento/usuario y contraseña) e inicio de sesión seguro para el acceso a la web administrativa

 |
| **Detalles:** | El Coordinador o Administrador ingresa sus credenciales de acceso al sistema. El Sistema verifica que el documento de identidad exista, que el rol del usuario requiera acceso administrativo (`requiere_password = true`) y que el hash de la contraseña coincida con `password_hash` en la base de datos para otorgar el acceso al panel de gestión.

 |
| **Eventos:** | **Ingresar Credenciales:** Captura del documento de identidad y la contraseña en la pantalla de inicio de sesión.

<br>

<br>**Validar Credenciales y Hash:** Búsqueda y comparación encriptada ejecutada por el Sistema en la base de datos.

<br>

<br>**Otorgar Acceso:** Inicia la sesión de usuario y redirige al panel administrativo correspondiente según el rol[cite: 1, 6, 9].<br>

<br>**Cerrar Sesión:** Destruye el token/sesión activa y devuelve al usuario a la pantalla de inicio de sesión[cite: 1, 9]. |
| **Extensiones:** | **1. Ingresar Credenciales**<br>

<br>1.1 El actor ingresa su documento de identificación y contraseña en el formulario de acceso web de gestión.

<br>

<br>1.2 Presiona el botón "Iniciar Sesión".

<br>

<br>

<br>**2. Validar Credenciales y Permisos (Proceso del Sistema)**<br>

<br>2.1 El Sistema consulta la tabla `usuarios` filtrando por el documento de identidad y verifica que la cuenta esté activa (`activo = true`).

<br>

<br>2.2 Consulta la tabla `roles` asociada al usuario para verificar que el perfil tenga permisos de gestión (`requiere_password = true`).

<br>

<br>2.3 Contrasta la clave en texto plano ingresada con la contraseña encriptada almacenada (`password_hash`).

<br>

<br>

<br>**3. Manejo de Errores de Autenticación**<br>

<br>3.1 Si el documento de identidad no existe o la contraseña es incorrecta, el Sistema cancela la autenticación y muestra la alerta: "Credenciales de acceso inválidas"[cite: 1, 9].<br>

<br>3.2 Si el rol asignado al usuario no tiene habilitado el acceso administrativo (`requiere_password = false`), el Sistema notifica: "El usuario no cuenta con privilegios para acceder al panel de gestión".

<br>

<br>3.3 Si la cuenta del usuario se encuentra inactiva (`activo = false`), el Sistema bloquea el ingreso e indica: "Su cuenta se encuentra inhabilitada. Contacte al administrador".

<br>

<br>

<br>**4. Otorgar Acceso y Redirección**<br>

<br>4.1 Al validar exitosamente las credenciales, el Sistema genera el token de sesión seguro.<br>

<br>4.2 Redirige al actor al panel o menú de administración correspondiente a sus atributos de rol[cite: 1, 6, 9].<br>

<br>

<br>**5. Cerrar Sesión**<br>

<br>5.1 El actor selecciona la opción "Cerrar Sesión" del menú de usuario.

<br>

<br>5.2 El Sistema invalida la sesión/token actual y redirige a la pantalla pública de login.
