### Cuadro de Escenario

| Componente | Especificación |
| --- | --- |
| **Nombre:** | Gestionar Usuarios |
| **Actores:** | Coordinador / Administrador

 |
| **Actividades:** | Registro, actualización, asignación de roles, departamentos, horarios y estado de la cuenta de usuario

 |
| **Detalles:** | El Coordinador o Administrador se encarga de dar de alta, consultar, editar o desactivar a los integrantes del sistema (técnicos contratados, programadores, pasantes y coordinadores), asignándoles su rol específico, departamento correspondiente y plantilla de horario asignada.

 |
| **Eventos:** | **Agregar Usuario:** Despliega el formulario emergente para ingresar un nuevo integrante.

**Guardar:** Almacena los datos personales, vinculación de departamento, rol y horario en la base de datos (`usuarios`).

**Cerrar:** Oculta el formulario sin conservar los datos.


**Filtrar:** Realiza búsquedas por documento de identidad, nombre, apellido, rol o departamento.

**Acciones:** Despliega las opciones de Editar y Desactivar/Borrar.


**Editar:** Abre la planilla para modificar los datos del usuario.

**Desactivar/Borrar:** Inactiva la cuenta del usuario o la elimina si no tiene historial.


**Primero, Anterior, Siguiente y Último:** Permite la navegación entre registros de la tabla.

 |
| **Extensiones:** | **1. Agregar Usuario**

1.1 Despliega la pantalla emergente solicitando el documento de identidad, nombres, apellidos, rol, departamento y la plantilla de horario asignada.

1.2 Si el rol seleccionado exige contraseña (`requiere_password = true`), habilita el campo para establecer la contraseña inicial.


**2. Guardar**

2.1 Invoca el proceso de verificación para validar que el documento de identificación no se encuentre duplicado.

2.2 Si existe duplicidad o faltan campos obligatorios, marca en rojo el campo afectado y despliega una alerta.

2.3 Si los datos son correctos, inserta el nuevo usuario en la tabla `usuarios` y actualiza la lista en pantalla.


**3. Cerrar**

3.1 Cancela el proceso y oculta el formulario emergente.


**4. Filtrar**

4.1 Realiza la consulta en la tabla `usuarios` aplicando el filtro deseado.

4.2 Muestra los resultados coincidentes en pantalla.

4.3 Si no existen coincidencias, la tabla se despliega vacía.


**5. Acciones**

5.1 Selecciona un usuario del listado y habilita los botones Editar y Desactivar/Borrar.


**6. Editar**

6.1 Carga los datos del usuario seleccionado para su modificación.

6.2 Verifica que los datos modificados cumplan con las reglas de negocio y de no duplicidad de documento.

6.3 Guarda las modificaciones en la base de datos y refresca la interfaz.


**7. Desactivar / Borrar**

7.1 Verifica si el usuario posee registros históricos en la tabla de `asistencias`.

7.2 Si el usuario registra asistencias previas, el sistema no lo elimina físicamente y realiza un borrado lógico estableciendo `activo = false` para garantizar la inmutabilidad de la auditoría.

7.3 Si el usuario no tiene registros de asistencia asociados, muestra una confirmación de alerta y elimina la cuenta de la base de datos.

**8. Primero, Anterior, Siguiente y Último**

8.1 Si no existen registros, los botones de navegación permanecen deshabilitados.

8.2 Si existen más de 10 usuarios registrados, se habilitan los botones Siguiente y Último.

8.3 Si la vista se desplaza a una página posterior a la primera, los botones Primero y Anterior se habilitan.
