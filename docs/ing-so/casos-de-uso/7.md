### Cuadro de Escenario

| Componente | Especificación |
| --- | --- |
| **Nombre:** | Generar Código QR de Identificación |
| **Actores:** | Coordinador / Administrador, Sistema (Generador de Credenciales)

 |
| **Actividades:** | Generación automática de token/hash único, codificación y renderizado del código QR como credencial física/digital

 |
| **Detalles:** | Al momento de guardar un nuevo usuario o a través de la opción de regeneración de credencial, el Sistema genera de forma automática un identificador/hash único invulnerable, construye la matriz del código QR y lo asocia en la base de datos (`codigo_qr`) al perfil del integrante para habilitar la autenticación alternativa de asistencia.

 |
| **Eventos:** | **Solicitar Generación de QR:** Disparo automático durante el alta de un usuario o acción manual de regeneración desde la gestión de perfil.

**Generar Hash Único:** Algoritmo ejecutado por el Sistema para crear una cadena única de identificación.

**Almacenar y Renderizar QR:** Guarda la referencia en la base de datos y dibuja el código en pantalla.

**Descargar / Imprimir QR:** Opción para exportar el gráfico en formato imprimible/PDF o descargar como imagen para entregárselo al usuario[cite: 1, 5, 9]. |
| **Extensiones:** | **1. Generar Hash Único (Proceso del Sistema)**

1.1 Al confirmar el registro de un usuario, el Sistema genera un token o hash único encriptado combinando los datos del usuario e identificadores del sistema.

1.2 El Sistema verifica en la tabla `usuarios` que el hash generado no coincida con ningún otro código previo (`codigo_qr = UNIQUE`).

1.3 Si se detectara colisión, el Sistema regenera la cadena hasta garantizar un valor 100% único.


**2. Almacenar y Renderizar QR**

2.1 El Sistema guarda la cadena del hash resultante en el campo `codigo_qr` de la tabla `usuarios`.

2.2 Genera la imagen bidimensional del código QR y la muestra en el módulo de detalle del usuario o en la pantalla de confirmación de registro.


**3. Descargar / Imprimir Credencial QR**

3.1 El actor selecciona la opción "Descargar Credencial QR" o "Imprimir Carnet"[cite: 1, 5, 9].

3.2 El Sistema abre un documento en formato PDF/imagen que incluye el nombre, apellido, documento de identidad, departamento, rol y el código QR generado del usuario, listo para su guardado o impresión[cite: 1, 2, 6, 9].


**4. Regeneración por Pérdida o Extravío**

4.1 Si el usuario reporta el extravío de su credencial QR, el Administrador o Coordinador presiona la opción "Regenerar Código QR".

4.2 El Sistema invalida el hash antiguo en la base de datos, genera una nueva clave única y reemplaza el registro previo en la tabla `usuarios` (`codigo_qr`).
