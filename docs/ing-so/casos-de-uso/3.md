### Cuadro de Escenario

| Componente | Especificación |
| --- | --- |
| **Nombre:** | Gestionar y Validar Rangos de Red Autorizados |
| **Actores:** | Coordinador / Administrador, Sistema (Verificador)

 |
| **Actividades:** | Parametrización y validación automática de direcciones IP o subredes de conexión

 |
| **Detalles:** | El Coordinador o Administrador se encarga de parametrizar y mantener la lista blanca de direcciones IP o notaciones CIDR autorizadas (`redes_autorizadas`). De forma automatizada, el Sistema valida la dirección IP del dispositivo cliente antes de procesar cualquier intento de marcación de asistencia.

 |
| **Eventos:** | **Agregar Red:** Despliega el formulario para añadir una nueva dirección IP o subred.

<br>

<br>**Guardar:** Registra la IP, su descripción y estado de activación en la base de datos[cite: 1, 2, 6].<br>

<br>**Cerrar:** Cancela la operación y cierra la ventana emergente.

<br>

<br>**Filtrar:** Permite realizar búsquedas por IP, descripción o estado.

<br>

<br>**Acciones:** Despliega las opciones de Editar, Activar/Desactivar y Eliminar.

<br>

<br>**Validar IP:** Comprobación automática ejecutada por el sistema al intentar marcar asistencia.

<br>

<br>**Primero, Anterior, Siguiente y Último:** Opciones para la navegación en la tabla.

 |
| **Extensiones:** | **1. Agregar Red**<br>

<br>1.1 Despliega el formulario emergente para ingresar la dirección IP (o notación CIDR) y la descripción del punto físico de conexión[cite: 1, 2, 6].<br>

<br>

<br>**2. Guardar**<br>

<br>2.1 Verifica que el formato de la dirección IP/CIDR sea válido (IPv4 o IPv6).<br>

<br>2.2 Comprueba que la dirección IP no se encuentre previamente registrada en la base de datos.<br>

<br>2.3 Si la IP es inválida o está duplicada, resalta el campo en rojo y muestra una alerta.

<br>

<br>2.4 Si los datos son válidos, almacena el registro activo en la tabla `redes_autorizadas`[cite: 1, 2, 6].<br>

<br>

<br>**3. Cerrar**<br>

<br>3.1 Oculta la ventana emergente sin guardar los cambios.

<br>

<br>

<br>**4. Filtrar**<br>

<br>4.1 Realiza la consulta en la tabla de redes autorizadas según el criterio ingresado[cite: 1, 2, 6].<br>

<br>4.2 Si existen coincidencias, las despliega en pantalla.

<br>

<br>4.3 Si no existen resultados, muestra la tabla vacía.

<br>

<br>

<br>**5. Acciones**<br>

<br>5.1 Selecciona un registro de red y habilita los botones Editar, Activar/Desactivar o Eliminar.

<br>

<br>

<br>**6. Editar**<br>

<br>6.1 Carga la información del rango de red seleccionado para su modificación.

<br>

<br>6.2 Executa las validaciones de formato e inalterabilidad de duplicados.<br>

<br>6.3 Actualiza la información en la base de datos y refresca la lista.

<br>

<br>

<br>**7. Validar IP de Marcación (Proceso Automático del Sistema)**<br>

<br>7.1 El cliente envía la solicitud de asistencia adjuntando la IP del dispositivo.

<br>

<br>7.2 El Sistema consulta la tabla `redes_autorizadas` filtrando por reglas activas (`activo = true`).

<br>

<br>7.3 Si la IP del cliente coincide con alguna regla válida, el Sistema autoriza la solicitud y permite continuar con la marcación.

<br>

<br>7.4 Si la IP no pertenece a la red autorizada o la regla está inactiva, el Sistema bloquea el proceso, muestra una alerta de "Red no autorizada" y finaliza la transacción.

<br>

<br>

<br>**8. Primero, Anterior, Siguiente y Último**<br>

<br>8.1 Si no existen datos, los botones de navegación permanecen deshabilitados.

<br>

<br>8.2 Si existen más de 10 registros, los botones Siguiente y Último se habilitan.

<br>

<br>8.3 Si la vista se encuentra en una página posterior a la primera, los botones Primero y Anterior se habilitan.
