### Cuadro de Escenario

|  |  |
| --- | --- |
| **Nombre:** | Validar Registro Único (Impedir Duplicidad) |
| **Actores:** | Coordinador / Administrador, Sistema (Verificador) |
| **Actividades:** | Comprobación y validación automática de unicidad de los datos de identificación de los usuarios |
| **Detalles:** | El Sistema ejecuta de forma automatizada una verificación en tiempo real sobre la tabla `usuarios` al momento de registrar o editar un integrante, validando que el documento de identidad (`documento_identidad`) no exista previamente para evitar cuentas duplicadas en la institución. |
| **Eventos:** | **Ingresar Documento de Identidad:** Captura de la cédula o pasaporte en la planilla de datos personales. |
| | **Consultar Unicidad:** Búsqueda automática ejecutada por el Sistema en la base de datos. |
| | **Aprobar Registro:** Habilita el guardado si no existen coincidencias previas. |
| | **Rechazar Registro:** Emite una alerta de duplicidad y bloquea el almacenamiento. |
| **Extensiones:** | **1. Consultar Unicidad (Proceso Automático del Sistema)** 1.1 Al perder el foco el campo de documento o al presionar "Guardar", el Sistema consulta la tabla `usuarios` filtrando por el valor de `documento_identidad`. 1.2 En creaciones de usuario, realiza una búsqueda global en la base de datos. En modificaciones, realiza la búsqueda excluyendo el registro del usuario en edición (`id != usuario_actual_id`). 1.3 Si no detecta registros coincidentes, el Sistema aprueba el valor y habilita la transacción. |
| | **2. Rechazar Registro (Manejo de Duplicidad)** 2.1 Si el Sistema detecta una coincidencia en la base de datos, cancela inmediatamente la acción de guardado. 2.2 El Sistema resalta en color rojo el campo de documento de identidad y muestra un mensaje informativo: "El documento de identidad ingresado ya se encuentra registrado en la plataforma". 2.3 Mantiene inhabilitada la opción de guardado definitivo hasta que el actor ingrese un documento válido y no registrado o cancele la operación. |
| | **3. Validación de Formato de Identificación** 3.1 El Sistema comprueba que el documento introducido contenga exclusivamente un formato alfanumérico válido según los estándares institucionales. 3.2 Si el formato es erróneo o incompleto, resalta el campo y solicita su corrección antes de realizar la consulta en la base de datos. |
