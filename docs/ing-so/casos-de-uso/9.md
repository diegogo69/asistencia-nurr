### Cuadro de Escenario

| Componente | Especificación |
| --- | --- |
| **Nombre:** | Registrar y Validar Asistencias |
| **Actores:** | Integrante (Técnico, Programador, Pasante), Sistema (Validador y Calculador)

 |
| **Actividades:** | Comprobación de red local autorizada, autenticación del usuario mediante biometría facial o código QR, determinación automática del tipo de evento (entrada/salida) y cálculo del estado de cumplimiento respecto al horario asignado.

 |
| **Detalles:** | El integrante (técnico, programador o pasante) realiza la marcación de su jornada en el punto de acceso. El Sistema verifica que la solicitud provenga de una IP autorizada, exige el documento de identidad y autentica al usuario mediante reconocimiento facial o lectura de su código QR. Luego, contrasta la fecha y hora actual contra el registro del día y la plantilla de horario asignada para determinar si es entrada o salida, calcular el estado (`puntual`, `retraso`, `anticipada`) y guardar la transacción en la base de datos (`asistencias`).

 |
| **Eventos:** | **Validar IP:** Comprobación automática de la dirección IP cliente en la lista blanca (`redes_autorizadas`).


**Ingresar Documento:** Captura de la cédula o pasaporte del usuario en la pantalla de marcación.


**Autenticar Facial / QR:** Verificación de identidad en tiempo real mediante el vector facial o el escaneo del código QR.


**Calcular Estado y Guardar Marcación:** Asignación automática del tipo de evento (entrada o salida), cálculo del cumplimiento de la jornada y almacenamiento persistente en la tabla `asistencias`.

 |
| **Extensiones:** | **1. Validar IP de Red (Proceso del Sistema)**

1.1 El Sistema obtiene la dirección IP desde donde se realiza la marcación.

1.2 Comprueba si la IP está registrada y activa en `redes_autorizadas`.

1.3 Si la IP no pertenece a la red autorizada, bloquea el inicio de la marcación y muestra la alerta: "Dispositivo fuera del rango de red autorizado".


**2. Ingresar Documento y Verificar Usuario**

2.1 El integrante ingresa su documento de identidad.

2.2 El Sistema busca la cédula en la tabla `usuarios` y verifica que la cuenta se encuentre activa (`activo = true`).

2.3 Si el usuario no existe o está inactivo, muestra la alerta: "Usuario no registrado o inhabilitado" y detiene la marcación[cite: 1, 2, 6].


**3. Autenticación de Identidad (Facial o QR)**

3.1 El Sistema activa la cámara web para proceder con la validación biométrica facial.

3.2 Compara el rostro capturado en tiempo real con la firma matemática almacenada (`vector_biometrico`).

3.3 **Mecanismo de Contingencia / Fallo Biométrico:** Si la biometría facial falla (baja iluminación, indisponibilidad del sensor), el Sistema habilita automáticamente el escaneo de credencial mediante el código QR como método alternativo.

3.4 Si la validación (facial o QR) es fallida o no coincide, el Sistema cancela el intento y muestra la alerta: "Autenticación de identidad fallida".


**4. Calcular Estado y Registrar Marcación**

4.1 El Sistema consulta la tabla `asistencias` para la fecha actual y el ID del usuario.

4.2 **Si no existe registro de entrada previo:** Registra la `hora_entrada`, la `ip_registro_entrada` y el `metodo_validacion` utilizado ('facial' o 'qr').

4.3 **Si ya existe un registro de entrada sin hora de salida:** Registra la `hora_salida` e `ip_registro_salida` en la misma fila del registro del día.

4.4 El Sistema contrasta la hora de la marcación con el horario asignado al usuario (`horarios_plantillas` / `horarios_detalles`).

4.5 Asigna el valor del `estado_cumplimiento` (`puntual`, `retraso` o `anticipada`).

4.6 Muestra en pantalla el mensaje de confirmación: "Marcación registrada exitosamente" con el detalle de hora y estado calculado.
