### Cuadro de Escenario

| Componente | Especificación |
| --- | --- |
| **Nombre:** | Consultar Historial de Asistencia |
| **Actores:** | Coordinador / Administrador, Sistema (Consultor Centralizado)

 |
| **Actividades:** | Búsqueda, filtrado dinámico multinivel y consulta de eventos históricos de marcación

 |
| **Detalles:** | El Coordinador o Administrador accede a la consulta histórica de la plataforma para visualizar los eventos de marcación de entrada y salida almacenados de forma inmutable en la base de datos (`asistencias`), aplicando filtros combinados por rango de fechas, usuario, departamento y estado de cumplimiento.

 |
| **Eventos:** | **Seleccionar Filtros:** Selección de parámetros de búsqueda (fecha inicio/fin, usuario, departamento o estado).

**Consultar Asistencias:** Búsqueda ejecutada por el Sistema cruzando las tablas de transacciones y perfiles.

**Renderizar Tabla de Resultados:** Despliegue en pantalla de las marcaciones coincidentes con su respectivo estado de cumplimiento y método de validación.

**Primero, Anterior, Siguiente y Último:** Controles de navegación para recorrer la tabla de resultados de forma paginada.

 |
| **Extensiones:** | **1. Seleccionar Filtros**

1.1 El actor ingresa a la sección "Historial de Asistencia".

1.2 Selecciona uno o varios criterios de filtrado: rango de fechas (fecha inicial y fecha final), usuario específico, departamento o estado de cumplimiento (`puntual`, `retraso`, `anticipada`, `incompleto`).


**2. Consultar Asistencias (Proceso del Sistema)**

2.1 El Sistema realiza la consulta sobre la tabla `asistencias` vinculando la información de las tablas `usuarios` y `departamentos` mediante sus claves foráneas.

2.2 Si no se aplican filtros específicos, el Sistema carga por defecto los registros correspondientes al día o jornada actual.

2.3 Si existen coincidencias en la base de datos, las ordena cronológicamente y las muestra en la tabla de resultados.

2.4 Si no existen registros que cumplan con los criterios seleccionados, el Sistema muestra la tabla vacía con el mensaje: "No se encontraron registros de asistencia para los filtros especificados".


**3. Visualización y Paginación**

3.1 Para cada registro mostrado, el Sistema presenta: fecha, nombre y apellido del usuario, departamento, hora de entrada, hora de salida, método utilizado ('facial' o 'qr') y estado de cumplimiento calculado.

3.2 **Primero, Anterior, Siguiente y Último:** Si no se retornan datos, los botones de paginación permanecen deshabilitados. Si la consulta genera más de 10 registros, se habilitan los botones Siguiente y Último. Si el usuario navega a partir de la segunda página, los botones Primero y Anterior se habilitan.
