### Cuadro de Escenario

|  |  |
| --- | --- |
| **Nombre:** | Detectar y Resolver Asistencias Incompletas |
| **Actores:** | Coordinador, Sistema (Detector y Registrador de Auditoría) |
| **Actividades:** | Detección automática de marcaciones sin registro de salida, notificación de anomalías, evaluación de justificaciones y registro de corrección en la bitácora de auditoría. |
| **Detalles:** | Al finalizar la jornada o al iniciar el turno operativo siguiente, el Sistema analiza los registros de la tabla `asistencias` identificando aquellos con `hora_entrada` procesada pero con `hora_salida` nula (`NULL`), cambiando automáticamente su estado a `incompleto` o `pendiente`. El Coordinador accede al panel dedicado de incidencias, evalúa la causa de la omisión (ej. olvido del usuario o constancia presencial), ingresa una justificación obligatoria y aprueba el nuevo estado de la asistencia, generando un registro inmutable en la tabla de auditoría (`asistencias_pendientes`). |
| **Eventos:** | **Detectar Registros Incompletos:** Proceso automático del Sistema que busca registros sin salida al cierre del turno. |
| | **Consultar Panel de Pendientes:** Filtrado y visualización de registros con marcaciones faltantes o inconsistentes. |
| | **Resolver / Justificar:** Apertura de la planilla de edición manual para imputar hora de salida, seleccionar el nuevo estado e ingresar la observación justificativa. |
| | **Registrar Auditoría:** Almacenamiento del historial de modificación en la tabla `asistencias_pendientes` y actualización del estado en `asistencias`. |
| **Extensiones:** | **1. Detectar Registros Incompletos (Proceso Automático del Sistema)** 1.1 El Sistema ejecuta una tarea automática al cumplirse el horario límite o cambio de fecha. 1.2 Revisa la tabla `asistencias` buscando entradas que no tengan registrada la `hora_salida` (`hora_salida IS NULL`). 1.3 Asigna automáticamente el valor 'incompleto' al campo `estado_cumplimiento` y notifica la anomalía al Coordinador en su panel de control. |
| | **2. Consultar Panel de Pendientes** 2.1 El Coordinador ingresa a la sección "Panel de Asistencias Incompletas". 2.2 El Sistema presenta una tabla con las marcaciones irregulares, detallando nombre del usuario, departamento, fecha y hora de entrada registrada. |
| | **3. Resolver y Justificar Inconsistencia** 3.1 El Coordinador selecciona un registro y presiona la opción "Resolver Incidencia". 3.2 El Sistema despliega una ventana emergente que solicita ingresar la hora de salida correspondiente, el nuevo estado de cumplimiento (`puntual`, `retraso`, etc.) y la `observacion` (justificación escrita). 3.3 Si el Coordinador intenta guardar sin redactar la observación justificativa, el Sistema resalta el campo en rojo y bloquea la confirmación indicando: "La observación explicativa es obligatoria para guardar la corrección". |
| | **4. Registrar Auditoría y Actualizar Estado** 4.1 Al ingresar datos válidos, el Sistema guarda un nuevo registro en la tabla `asistencias_pendientes` asociando el ID de la asistencia (`asistencia_id`), el ID del coordinador (`coordinador_id`), la nota explicativa (`observacion`), la fecha y hora de resolución, el estado anterior ('incompleto') y el nuevo estado aprobado. 4.2 Actualiza el registro en la tabla `asistencias` imputando la hora de salida acordada y modificando el `estado_cumplimiento` al nuevo valor autorizado. 4.3 Despliega el mensaje: "Inconsistencia resuelta y registrada en auditoría exitosamente" y remueve el elemento de la lista de pendientes. |
