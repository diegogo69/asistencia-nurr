### Cuadro de Escenario

| Componente | Especificación |
| --- | --- |
| **Nombre:** | Gestionar Unidades Organizacionales |
| **Actores:** | Coordinador / Administrador

 |
| **Actividades:** | Configuración y parametrización de la estructura organizativa

 |
| **Detalles:** | El Coordinador o Administrador se encargará de crear, consultar, actualizar o eliminar las unidades organizacionales institucionales (Dirección, Coordinación, Sección, Oficina) dentro del sistema para mantener actualizada la estructura de la ULA-NURR.

 |
| **Eventos:** | **Agregar Unidad:** Despliega una ventana/planilla emergente para registrar una nueva unidad organizacional.


**Guardar:** Almacena los datos de la unidad en la base de datos (`departamentos`)

**Cerrar:** Cierra la ventana emergente.


**Filtrar:** Realiza búsquedas en la tabla por nombre o tipo de unidad.


**Acciones:** Despliega las opciones de Editar y Eliminar.

**Editar:** Permite modificar el nombre o clasificación de una unidad seleccionada.

**Eliminar:** Permite remover o desactivar una unidad de la base de datos.


**Primero, Anterior, Siguiente y Último:** Opciones para navegar en la tabla de registros.

 |
| **Extensiones:** | **1. Agregar Unidad**

1.1 Despliega la planilla emergente para ingresar el nombre de la unidad y seleccionar su tipo (Dirección, Coordinación, Sección, Oficina).


**2. Guardar**
2.1 Verifica que el nombre de la unidad organizacional no esté duplicado en la base de datos.

2.2 Si existe una unidad duplicada, resalta el campo en rojo y muestra una alerta.

2.3 Si los datos son correctos, guarda el registro en la base de datos y actualiza la tabla.


**3. Cerrar**
3.1 Oculta la pantalla emergente sin guardar cambios.


**4. Filtrar**

4.1 Busca la existencia de la unidad en la base de datos.

4.2 Compara la información en la base de datos.

4.3 Si existe, muestra las coincidencias en pantalla.

4.4 Si no existe, muestra la tabla vacía.


**5. Acciones**

5.1 Selecciona el registro de la unidad y habilita los botones Editar y Eliminar.


**6. Editar**

6.1 Busca los datos de la unidad organizacional a modificar.

6.2 Verifica que los datos modificados sean correctos y no generen duplicidad.

6.3 Actualiza los datos en la base de datos y refresca la vista.


**7. Eliminar**

7.1 Muestra una alerta para confirmar la eliminación de la unidad.

7.2 Verifica si la unidad tiene usuarios o dependencias vinculadas.

7.3 Si existen dependencias, notifica al usuario que no se puede eliminar la unidad.

7.4 Si no existen dependencias, elimina o desactiva la unidad de la base de datos.


**8. Primero, Anterior, Siguiente y Último**

8.1 Si no existen datos, estos botones estarán deshabilitados.

8.2 Si existen más de 10 registros, los botones Siguiente y Último estarán habilitados.

8.3 Si se visualizan registros posteriores a la primera página, los botones Primero y Anterior estarán habilitados.
