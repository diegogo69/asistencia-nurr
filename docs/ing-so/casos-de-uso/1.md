### Cuadro de Escenario

| Componente | Especificación |
| --- | --- |
| **Nombre:** | Gestionar Unidades Organizacionales |
| **Actores:** | Coordinador / Administrador

 |
| **Actividades:** | Configuración y parametrización de la estructura organizativa

 |
| **Detalles:** | El Coordinador o Administrador se encargará de crear, consultar, actualizar o eliminar las unidades organizacionales institucionales (Dirección, Coordinación, Sección, Oficina) dentro del sistema para mantener actualizada la estructura de la ULA-NURR.

 |
| **Eventos:** | **Agregar Unidad:** Despliega una ventana/planilla emergente para registrar una nueva unidad organizacional.

<br>

<br>**Guardar:** Almacena los datos de la unidad en la base de datos (`departamentos`)[cite: 1, 6].<br>

<br>**Cerrar:** Cierra la ventana emergente.

<br>

<br>**Filtrar:** Realiza búsquedas en la tabla por nombre o tipo de unidad.

<br>

<br>**Acciones:** Despliega las opciones de Editar y Eliminar.

<br>

<br>**Editar:** Permite modificar el nombre o clasificación de una unidad seleccionada.

<br>

<br>**Eliminar:** Permite remover o desactivar una unidad de la base de datos.

<br>

<br>**Primero, Anterior, Siguiente y Último:** Opciones para navegar en la tabla de registros.

 |
| **Extensiones:** | **1. Agregar Unidad**<br>

<br>1.1 Despliega la planilla emergente para ingresar el nombre de la unidad y seleccionar su tipo (Dirección, Coordinación, Sección, Oficina).

<br>

<br>

<br>**2. Guardar**<br>

<br>2.1 Verifica que el nombre de la unidad organizacional no esté duplicado en la base de datos.<br>

<br>2.2 Si existe una unidad duplicada, resalta el campo en rojo y muestra una alerta.

<br>

<br>2.3 Si los datos son correctos, guarda el registro en la base de datos y actualiza la tabla.

<br>

<br>

<br>**3. Cerrar**<br>

<br>3.1 Oculta la pantalla emergente sin guardar cambios.

<br>

<br>

<br>**4. Filtrar**<br>

<br>4.1 Busca la existencia de la unidad en la base de datos.

<br>

<br>4.2 Compara la información en la base de datos.

<br>

<br>4.3 Si existe, muestra las coincidencias en pantalla.

<br>

<br>4.4 Si no existe, muestra la tabla vacía.

<br>

<br>

<br>**5. Acciones**<br>

<br>5.1 Selecciona el registro de la unidad y habilita los botones Editar y Eliminar.

<br>

<br>

<br>**6. Editar**<br>

<br>6.1 Busca los datos de la unidad organizacional a modificar.

<br>

<br>6.2 Verifica que los datos modificados sean correctos y no generen duplicidad.

<br>

<br>6.3 Actualiza los datos en la base de datos y refresca la vista.

<br>

<br>

<br>**7. Eliminar**<br>

<br>7.1 Muestra una alerta para confirmar la eliminación de la unidad.

<br>

<br>7.2 Verifica si la unidad tiene usuarios o dependencias vinculadas.<br>

<br>7.3 Si existen dependencias, notifica al usuario que no se puede eliminar la unidad.<br>

<br>7.4 Si no existen dependencias, elimina o desactiva la unidad de la base de datos.

<br>

<br>

<br>**8. Primero, Anterior, Siguiente y Último**<br>

<br>8.1 Si no existen datos, estos botones estarán deshabilitados.

<br>

<br>8.2 Si existen más de 10 registros, los botones Siguiente y Último estarán habilitados.

<br>

<br>8.3 Si se visualizan registros posteriores a la primera página, los botones Primero y Anterior estarán habilitados.
